## ⚠️ Minor Issues (Non-blocking)

### 1. `normalizedNews` mutates articles in place conceptually
```typescript
category: n.category?.split('|')[0]?.trim() || 'Outros',
```
This creates new objects (spread `...n` is correct), so no mutation. But if other parts of the codebase rely on `allNews[i].category` having the original pipe-separated value for display purposes (e.g., showing "tags"), they now see the truncated value. **Risk: low** — the only consumer verified is `filteredNews` and the sidebar.

### 2. `categories` useMemo has empty deps array
```typescript
const categories = useMemo(() => ['Todos', ...CANONICAL_CATEGORIES], []);
```
Correct since `CANONICAL_CATEGORIES` is a module-level constant. No issue.

### 3. `limit=2000` in Friday news.js
Loading 2000 articles at once into browser memory with no pagination. At ~2KB per article object = ~4MB DOM payload. Acceptable for now but will degrade with growth. **Recommendation**: add pagination when article count > 5000.

### 4. Test 2 is not a RED test
`Supabase has pipe-separated categories that need normalization` — this test validates that raw data HAS pipes (documenting the problem), not that the fix is applied. It will pass both before and after the fix. It's a data assertion, not a behavior assertion. **Non-blocking** — it's still useful as a data contract test.

### 5. `IA` alias not in sidebar
The `effectiveCategory = selectedCategory === 'IA' ? 'Tecnologia & IA' : selectedCategory` alias works for filtering, but there's no `'IA'` button in the sidebar for users to click. The sidebar only shows the 17 canonical names. Users would need to know to type/set `IA` programmatically. **Ambiguous requirement** — if user expects a visible `IA` shortcut button, that's not yet rendered.
